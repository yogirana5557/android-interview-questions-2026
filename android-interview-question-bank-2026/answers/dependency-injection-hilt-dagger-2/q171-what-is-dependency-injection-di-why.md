# Question 171: What is Dependency Injection (DI)? Why is it used?

**Category:** Dependency Injection (Hilt, Dagger 2)  
**Difficulty:** Junior

---

### Answer

Dependency Injection is a design pattern where an object's dependencies are provided to it by an external container or framework, rather than the object creating them itself. DI is used to achieve loose coupling, improve code reuse, make code easier to test (by injecting mocks), and manage object lifecycles automatically.
